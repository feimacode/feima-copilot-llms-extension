## 计费公式

每次请求使用以下简单公式计算：

```
加权请求次数 = 模型倍率 × 上下文倍率
```

结果保留两位小数。

### 计算示例

| 模型级别 | 上下文大小 | 模型倍率 | 上下文倍率 | 加权请求 |
|----------|------------|----------|------------|----------|
| 超轻量级 | 小上下文 (0-4K tokens) | 0.1x | 0.5x | 0.05 |
| 轻量级 | 中等上下文 (4K-16K tokens) | 0.5x | 1.0x | 0.50 |
| 标准 | 大上下文 (16K-32K tokens) | 1.0x | 1.5x | 1.50 |
| 高级 | 超大上下文 (32K+ tokens) | 2.0x | 2.0x | 4.00 |

## 模型倍率

不同的 AI 模型根据其能力和资源需求有不同的费用：

### 超轻量级模型 (0.1x)

**示例**: Qwen3 Flash

- **适用场景**: 超快速代码补全
- **特点**: 极快速度、极低资源占用
- **使用时机**: 需要即时响应的非常简单的任务

### 轻量级模型 (0.5x)

**示例**: Qwen3.5 Plus

- **适用场景**: 代码补全、简单代码生成
- **特点**: 快速、高效、资源占用低
- **使用时机**: 需要快速响应的简单任务

### 标准模型 (1.0x)

**示例**: Qwen3 Max、DeepSeek V3.2、Kimi K2.5

- **适用场景**: 通用问答、代码理解
- **特点**: 性能与成本平衡
- **使用时机**: 不需要高级推理的日常编程任务

### 高级模型 (2.0x)

**示例**: Qwen3 Coder Plus、GLM-5

- **适用场景**: 大型代码库分析、高级推理
- **特点**: 最先进性能、最大能力
- **使用时机**: 需要最高质量输出的关键任务

## 上下文倍率

更大的上下文窗口需要更多计算资源：

| 上下文大小 | Token 范围 | 倍率 |
|------------|------------|------|
| 小上下文 | 0 - 4,000 tokens | 0.5x |
| 中等上下文 | 4,001 - 16,000 tokens | 1.0x |
| 大上下文 | 16,001 - 32,000 tokens | 1.5x |
| 超大上下文 | 32,001+ tokens | 2.0x |

### 什么是 Token？

Token 是 AI 模型处理文本的基本单位：

- **英文**: 大约 1 token ≈ 4 个字符或 ¾ 个单词
- **中文**: 大约 1 token ≈ 1-2 个字符
- **代码**: Token 数量取决于编程语言和语法

**示例**: 短语 "Hello, world!" 大约是 4 个 token。

## 成本优化建议

### 1. 简单任务使用轻量级模型

```markdown
# 推荐：使用轻量级模型进行简单补全
"完成这个函数：def calculate_sum(a, b):"

# 避免：对相同任务使用高级模型
```

### 2. 保持上下文简洁

```markdown
# 推荐：聚焦上下文
"审查这个函数的 bug：[粘贴 20 行代码]"

# 避免：只需要函数时包含整个文件
```

### 3. 日常编程使用标准模型

大多数日常任务使用标准模型即可。仅在以下情况使用高级模型：

- 复杂算法设计
- 架构决策
- 关键系统的代码审查
- 大规模重构

### 4. 监控使用情况

定期检查余额：

1. 点击飞码状态栏项
2. 查看剩余请求次数
3. 在 [网页仪表板](https://feimacode.cn/profile) 查看交易历史

## 计费示例

### 示例 1：代码补全

**场景**: 使用 Qwen Turbo Lite 完成函数

- **模型**: Qwen Turbo Lite（超轻量级，0.1x）
- **上下文**: 2,000 tokens（小上下文，0.5x）
- **计算**: 0.1 × 0.5 = **0.05 加权请求**

### 示例 2：代码审查

**场景**: 使用 Qwen3 Max 审查复杂函数

- **模型**: Qwen3 Max（标准，1.0x）
- **上下文**: 8,000 tokens（中等上下文，1.0x）
- **计算**: 1.0 × 1.0 = **1.00 加权请求**

### 示例 3：大型代码库分析

**场景**: 使用 Qwen3 Coder Plus 分析整个模块

- **模型**: Qwen3 Coder Plus（高级，2.0x）
- **上下文**: 50,000 tokens（超大上下文，2.0x）
- **计算**: 2.0 × 2.0 = **4.00 加权请求**

## 常见问题

### 如何查看当前余额？

登录后，您可以在 [飞码网站](https://feimacode.cn/profile) 的个人中心页面查看余额和交易历史。

### 上下文区间会变化吗？

上下文区间配置可能会定期调整以反映计算成本。请查看此页面获取最新信息。

### 应该使用哪个模型？

大多数任务从标准模型开始。简单任务使用轻量级模型，仅在需要最高质量输出时使用高级模型。

### Token 如何计算？

Token 根据请求的总输入和输出计算。输入包括您的提示和提供的任何上下文。输出是模型的响应。

### 可以查看计费历史吗？

可以，您可以在 [飞码网站](https://feimacode.cn/profile) 查看完整的计费历史，包括：

- 交易时间戳
- 使用的模型
- Token 数量
- 加权请求成本
